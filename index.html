الخطوة الأولى — اسأل Claude يحلل الـ Docs

ارفع الـ BRD/SRS على Claude أو ChatGPT واعمل الـ prompt ده قبل أي حاجة تانية:

Prompt Claude / ChatGPT
تحليل الـ BRD/SRS
# ارفع الـ PDF/Word مع الـ prompt ده

أنا UI/UX Designer وبدأ أشتغل على المشروع ده.
حلّل الـ document ده واعمل ليا:

1. ملخص المشروع (5 جمل بالعربي)
   - ايه المشكلة اللي بتحلها؟
   - مين المستخدم الأساسي (Primary User)?
   - ايه الـ core actions اللي المستخدم هيعملها؟

2. قائمة الشاشات
   - اعملي جدول بكل screen مع:
     * اسم الشاشة
     * وظيفتها (1 جملة)
     * الـ user actions جوّاها
     * هل في auth؟ (public / private)

3. الـ User Flows الرئيسية
   - اكتب كل flow خطوة بخطوة
   - مثال: "Onboarding Flow: Splash → Login → Home"

4. نقاط التحذير للـ Designer
   - أي requirements ممكن تأثر على الـ UX
   - Edge cases مهمة لازم أراعيها

5. أسئلة ناقصة
   - أي معلومة مش موجودة في الـ doc محتاجها كـ designer

الخطوة التانية — استخرج الـ Information Architecture
Prompt Claude / ChatGPT
Information Architecture
بناءً على تحليلنا السابق، اعمل الـ Information Architecture للتطبيق:

1. Navigation Structure
   - ايه الـ bottom nav items؟ (لو mobile)
   - ايه الـ main sections؟
   - ايه الـ sub-pages لكل section؟

2. Screen Inventory Table
   Format:
   | Screen Name | Parent | Type | Priority |
   |-------------|--------|------|----------|
   
   Types: Onboarding / Auth / Main / Detail / Modal / Settings

3. اقترح أسماء الـ Frames في Figma
   بالـ naming convention: [section]/[screen]
   مثال: auth/login | home/feed | product/detail

4. أولويات التصميم
   رتّب الشاشات من الأهم للأقل أهمية للـ MVP

Phase 2 — قبل ما تفتح Figma
Research & Visual Direction
استخدم الـ AI تحدد الـ visual language اللي يناسب المشروع قبل ما تبدأ تصميم أي شاشة
🎨
حدد الـ Visual Direction مع Claude
Prompt Claude / ChatGPT
Visual Direction
أنا بصمم [نوع التطبيق] للـ [target audience].
المشروع ده [وصف مختصر من الـ BRD].

اقترح Visual Direction مناسب:

1. Personality Keywords (5-7 كلمة)
   مثال: Modern, Trustworthy, Clean, Energetic

2. Color Palette
   - Primary color (مع مبرر ليه هذا اللون)
   - Secondary color
   - Accent color
   - Background + Surface colors
   - Error / Success / Warning colors
   - اديني الـ HEX codes

3. Typography
   - Font للـ headings (من Google Fonts)
   - Font للـ body
   - Type scale مقترح (H1, H2, H3, Body, Caption)

4. Visual Style
   - Flat / Neumorphic / Glassmorphism / Material / Minimal
   - Corner radius المناسب (sharp 0px / rounded 8px / pill 24px)
   - Shadows: none / subtle / prominent
   - Illustrations أو Photos أو Icons بس؟

5. Competitor Analysis
   - اذكر 3 apps ممكن نأخذ منها inspiration
   - مع ايه اللي نأخذه وايه اللي نتجنبه

الـ Iteration Loop مع readdy
  • 1
    Generate الشاشة استخدم الـ full prompt template — مش prompt قصير
  • 2
    قيّم النتيجة بالـ criteria دول Hierarchy واضحة؟ Colors consistent؟ Spacing منتظم؟ المحتوى صح؟
  • 3
    Refine بـ follow-up prompt "Make the CTA button bigger" / "Add more spacing between sections" / "Change header style"
  • 4
    Export وادخل Figma استخدم الـ output كـ reference — مش final design
  • 5
    Rebuild في Figma بـ Components الـ readdy output مجرد direction — الـ Figma file هو الـ source of truth

Handoff Checklist النهائي
  • 🎨 Design System
  • Figma Variables معمولة (Colors, Typography, Spacing, Radius)
  • كل الـ text styles محددة وـ Auto-applied
  • مفيش hardcoded colors في أي screen
  • 🏷️ Naming & Structure
  • الـ Figma file فيه 4 pages: Design System / Components / Screens / Flows
  • كل الـ Screens مسماة بـ snake_case: auth/login, home/feed
  • كل الـ Layers مسماة (مفيش Frame 892)
  • كل الـ Components Figma Components (مش Groups)
  • 📐 Layout
  • Auto Layout مستخدم في كل الـ frames
  • مفيش absolute positioning (غير للعناصر الـ overlay)
  • الـ spacing من الـ Variables (مش hardcoded)
  • 🧩 Components
  • كل Component عنده Variants: default, hover, disabled, loading, error
  • الـ Component Properties موثّقة (text, icon, isEnabled)
  • Empty states مصمّمة لكل list/feed
  • Error states مصمّمة لكل form/request
  • 📝 Dev Annotations
  • كل شاشة عندها وصف وظيفتها
  • الـ interactive elements فيها navigation annotation
  • الـ API-dependent components فيها endpoint reference
  • 🖼️ Assets
  • الـ Icons exported كـ SVG
  • الـ Images placeholders واضحة
  • الـ App Icon في كل المقاسات
  • 🔗 Prototype
  • كل الـ User Flows متربطة في Prototype
  • الـ Navigation actions موضحة
  • Shareable Prototype link جاهز
