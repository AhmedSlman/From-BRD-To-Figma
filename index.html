# ارفع الـ PDF/Word مع الـ prompt ده

أنا UI/UX Designer وبدأ أشتغل على المشروع ده.
حلّل الـ document ده واعمل ليا:

1. ملخص المشروع (5 جمل بالعربي)
   - ايه المشكلة اللي بتحلها؟
   - مين المستخدم الأساسي (Primary User)?
   - ايه الـ core actions اللي المستخدم هيعملها؟

2. قائمة الشاشات
   - اعملي جدول بكل screen مع:
     * اسم الشاشة
     * وظيفتها (1 جملة)
     * الـ user actions جوّاها
     * هل في auth؟ (public / private)

3. الـ User Flows الرئيسية
   - اكتب كل flow خطوة بخطوة
   - مثال: "Onboarding Flow: Splash → Login → Home"

4. نقاط التحذير للـ Designer
   - أي requirements ممكن تأثر على الـ UX
   - Edge cases مهمة لازم أراعيها

5. أسئلة ناقصة
   - أي معلومة مش موجودة في الـ doc محتاجها كـ designer

حدد الـ Visual Direction مع Claude
Prompt Claude / ChatGPT
Visual Direction
أنا بصمم [نوع التطبيق] للـ [target audience].
المشروع ده [وصف مختصر من الـ BRD].

اقترح Visual Direction مناسب:

1. Personality Keywords (5-7 كلمة)
   مثال: Modern, Trustworthy, Clean, Energetic

2. Color Palette
   - Primary color (مع مبرر ليه هذا اللون)
   - Secondary color
   - Accent color
   - Background + Surface colors
   - Error / Success / Warning colors
   - اديني الـ HEX codes

3. Typography
   - Font للـ headings (من Google Fonts)
   - Font للـ body
   - Type scale مقترح (H1, H2, H3, Body, Caption)

4. Visual Style
   - Flat / Neumorphic / Glassmorphism / Material / Minimal
   - Corner radius المناسب (sharp 0px / rounded 8px / pill 24px)
   - Shadows: none / subtle / prominent
   - Illustrations أو Photos أو Icons بس؟

5. Competitor Analysis
   - اذكر 3 apps ممكن نأخذ منها inspiration
   - مع ايه اللي نأخذه وايه اللي نتجنبه

الـ Iteration Loop مع readdy
  • 1
    Generate الشاشة استخدم الـ full prompt template — مش prompt قصير
  • 2
    قيّم النتيجة بالـ criteria دول Hierarchy واضحة؟ Colors consistent؟ Spacing منتظم؟ المحتوى صح؟
  • 3
    Refine بـ follow-up prompt "Make the CTA button bigger" / "Add more spacing between sections" / "Change header style"
  • 4
    Export وادخل Figma استخدم الـ output كـ reference — مش final design
  • 5
    Rebuild في Figma بـ Components الـ readdy output مجرد direction — الـ Figma file هو الـ source of truth

Handoff Checklist النهائي
  • 🎨 Design System
  • Figma Variables معمولة (Colors, Typography, Spacing, Radius)
  • كل الـ text styles محددة وـ Auto-applied
  • مفيش hardcoded colors في أي screen
  • 🏷️ Naming & Structure
  • الـ Figma file فيه 4 pages: Design System / Components / Screens / Flows
  • كل الـ Screens مسماة بـ snake_case: auth/login, home/feed
  • كل الـ Layers مسماة (مفيش Frame 892)
  • كل الـ Components Figma Components (مش Groups)
  • 📐 Layout
  • Auto Layout مستخدم في كل الـ frames
  • مفيش absolute positioning (غير للعناصر الـ overlay)
  • الـ spacing من الـ Variables (مش hardcoded)
  • 🧩 Components
  • كل Component عنده Variants: default, hover, disabled, loading, error
  • الـ Component Properties موثّقة (text, icon, isEnabled)
  • Empty states مصمّمة لكل list/feed
  • Error states مصمّمة لكل form/request
  • 📝 Dev Annotations
  • كل شاشة عندها وصف وظيفتها
  • الـ interactive elements فيها navigation annotation
  • الـ API-dependent components فيها endpoint reference
  • 🖼️ Assets
  • الـ Icons exported كـ SVG
  • الـ Images placeholders واضحة
  • الـ App Icon في كل المقاسات
  • 🔗 Prototype
  • كل الـ User Flows متربطة في Prototype
  • الـ Navigation actions موضحة
  • Shareable Prototype link جاهز

Open Claude, make sure the Figma connector is connected, and send this prompt with your Figma file link:

Main Prompt Claude + Figma MCP
Behance Presentation — Figma MCP
# Paste your Figma file link here
Figma file: [YOUR FIGMA FILE URL]

I'm a UI designer and I just finished designing an app called [APP NAME].
I want you to build a Behance case study presentation as a new Figma page inside the same file.

Steps:
1. Open the Figma file and read all existing screens
2. Extract: primary colors, fonts, and the 4–6 most important screens
3. Create a new page named "Behance Presentation"
4. Build these sections as Figma frames at 1440px width:

Frame 1 — HERO COVER (1440×810px)
- Background using the primary color from the design system
- App name in large bold type (use the project's heading font)
- Short tagline below the name
- Placeholder area for a phone mockup (labeled clearly)
- Designer name in the bottom corner

Frame 2 — PROJECT OVERVIEW (1440×600px)
- App name + 2–3 sentence project description
- 3 info cards side by side: Role | Tools | Year
- Clean card-style layout

Frame 3 — DESIGN SYSTEM (1440×700px)
- Color palette: each color in a labeled swatch with its HEX code
- Typography scale: font name + H1 / H2 / Body / Caption samples
- Corner radius and spacing tokens (if found in the file)

Frame 4 — SCREENS SHOWCASE (1440×1200px)
- Display 4–6 of the key screens
- Each screen inside a simple rounded-rectangle phone frame
- Screen name label below each one
- Dark background so the screens stand out

Frame 5 — KEY FEATURES (1440×700px)
- 3 or 4 standout features of the app
- Each feature: icon (emoji or simple shape) + title + one-line description
- Equal-width horizontal layout

Frame 6 — CLOSING (1440×500px)
- "Thank You" or a short closing line
- Designer name
- "Made with Figma" badge

Rules:
- Use the same fonts and colors from the project's design system — no hardcoded values
- Apply Auto Layout to every frame
- Consistent padding: 48px on all sides
- Clear layer names: behance/hero, behance/overview, behance/design-system, etc.

# Audit and fix
Review all frames in the "Behance Presentation" page.
Identify any spacing issues or unreadable text.
Then fix them directly in Figma.
🔄
Full Workflow — Step by Step
  • 1
    Connect Claude to Figma Claude.ai → Settings → Connectors → Figma → Connect. Then open a new chat.
  • 2
    Send the prompt with your Figma link Claude will open the file, read all screens, colors, and Design System automatically.
  • 3
    Claude builds the Presentation It creates a new "Behance Presentation" page with all frames directly inside your Figma file.
  • 4
    Review and iterate Open Figma to check the result. Send a follow-up prompt for any changes — Claude edits directly.
  • 5
    Export from Figma Select all frames in the Behance page → Export as PNG 2x (or 1x if file size is too large).
  • 6
    Upload to Behance New Project → Upload PNGs in order → Title + Description + Tags → Publish.
✅
Behance Upload Checklist
  • 🖼️ Figma Frames
  • Every frame is 1440px wide (Behance standard)
  • Order is correct: Cover → Overview → DS → Screens → Features → Closing
  • Colors and fonts come from the Design System — no hardcoded values
  • Layer names are clear: behance/hero, behance/screens
  • 📤 Export
  • PNG 2x — no frame smaller than 1440×600px
  • No frame has clipped text or overflow
  • Files named in order: 01-cover, 02-overview, etc.
  • 📝 Behance Project Info
  • Clear and distinct project title
  • Description: 2–3 sentences in English
  • Tags: UI, UX, Mobile App, Figma, [App Category]
  • Tools: Figma (required for Behance search visibility)
  • Cover Image: set to the Hero frame (the most striking one)
